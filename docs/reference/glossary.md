# Glossary

This section defines key terms used throughout the textbook.

## A

**Action**: In ROS 2, a communication pattern that supports long-running requests with feedback and result reporting.

**APA Format**: American Psychological Association style for citations and references, used throughout this textbook.

## B

**Behavior Tree**: A hierarchical structure used to represent and execute complex robot behaviors.

## C

**Citation**: A reference to a source used in the textbook, formatted according to APA style.

## D

**Digital Twin**: A virtual representation of a physical robot or system, used for simulation and testing.

## E

**Entity**: In the context of this textbook, refers to content components like modules, sections, code examples, etc.

## F

**Forward Kinematics**: The use of kinematic equations to compute the position of the end-effector from specified values of joint parameters.

## G

**Gazebo**: A robotics simulator providing physics simulation, sensor simulation, and visual rendering.

## I

**Inverse Kinematics**: The use of kinematic equations to compute joint parameters from the position of the end-effector.

**Isaac**: NVIDIA's simulation and computing platform for robotics and AI applications.

## L

**Learning Objective**: A clear, specific statement about what learners will be able to do after completing a module.

## M

**Module**: A major section of the textbook covering a specific topic area (e.g., ROS 2, Simulation, Isaac).

## N

**Node**: In ROS 2, a process that performs computation and communicates with other nodes.

## P

**Publisher**: In ROS 2, a node that sends messages to a topic.

**Perception**: The process of interpreting sensory information to understand the environment.

## R

**ROS 2**: Robot Operating System 2, a flexible framework for writing robot software.

## S

**Subscriber**: In ROS 2, a node that receives messages from a topic.

**Simulation**: The imitation of the operation of a real-world process or system over time.

## T

**Topic**: In ROS 2, a named bus over which nodes exchange messages.

## U

**Unity**: A real-time 3D development platform used for creating simulation environments.

## V

**VLA**: Vision-Language-Action, systems that integrate perception, language understanding, and action execution.

## W

**Workspace**: In ROS 2, a folder containing multiple packages intended to be built together.
